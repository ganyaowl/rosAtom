# ☢️ Radiation Monitor

Мобильное приложение для мониторинга уровня радиации по стране в реальном времени.
Стек: **React Native (Expo, TypeScript)** + **Node.js/Express** backend.

## Структура проекта

```
radiation-monitor/
├── backend/            # Express REST API (моковые данные)
│   ├── data/            # stations.json — базовые станции
│   ├── routes/          # /stations, /statistics, /alerts
│   ├── utils/           # генерация случайных показаний
│   └── server.js
└── mobile/              # Expo приложение (TypeScript)
    ├── App.tsx
    └── src/
        ├── api/          # HTTP-клиент и эндпоинты
        ├── components/   # GlassCard, StationCard, графики, badge и т.д.
        ├── hooks/        # useStations, useStatistics, useAlerts...
        ├── navigation/   # Tabs + Stack (React Navigation)
        ├── screens/      # Home, Map, Statistics, StationDetail, Notifications, Settings
        ├── services/     # stationService (с fallback на mock-данные)
        ├── theme/        # цвета, ThemeContext (тёмная/светлая тема, настройки)
        ├── types/        # общие TypeScript-типы
        └── utils/        # форматирование дат и уровней радиации
```

## 1. Запуск backend

```bash
cd backend
npm install
npm start
```

Сервер поднимется на `http://localhost:4000`. Проверьте в браузере:
- `GET /stations`
- `GET /stations/:id`
- `GET /statistics`
- `GET /alerts`

Данные генерируются случайно при каждом запросе (с небольшим кэшем в 5 сек для консистентности), чтобы демонстрировать "реальное время" на хакатоне.

## 2. Запуск мобильного приложения

```bash
cd mobile
npm install
npx expo start
```

Откройте приложение через Expo Go (отсканируйте QR-код) или запустите эмулятор:

```bash
npx expo start --android
# или
npx expo start --ios
```

### Важно: подключение к backend

Файл `mobile/src/api/client.ts` использует:
- `http://localhost:4000` — для iOS-симулятора и веб;
- `http://10.0.2.2:4000` — для Android-эмулятора.

Если вы тестируете на **физическом телефоне** через Expo Go, замените адрес на IP вашего компьютера в локальной сети, например:

```ts
export const API_BASE_URL = "http://192.168.1.23:4000";
```

(IP можно узнать командой `ipconfig` / `ifconfig`).

**Приложение спроектировано так, что если backend недоступен — экраны автоматически переключаются на локальные моковые данные**, поэтому демо не сломается, даже если сервер не запущен или недоступен по сети.

### Карты (react-native-maps)

- На iOS карта работает "из коробки" (Apple Maps).
- На Android для `PROVIDER_GOOGLE` нужен Google Maps API key — вставьте его в `mobile/app.json` → `android.config.googleMaps.apiKey`. Без ключа карта на Android может не отобразиться корректно в standalone-сборке (в Expo Go обычно работает с ограничениями).

## 3. Экраны приложения

| Экран | Описание |
|---|---|
| **Главная** | Средний уровень радиации по стране, индикатор статуса (🟢🟡🟠🔴), мини-карта, дата обновления, кнопка "Обновить", список последних станций |
| **Карта** | Полноэкранная карта со всеми станциями и тепловыми зонами; тап по точке открывает карточку со сведениями |
| **Статистика** | Графики изменения уровня, среднее/макс/мин, недельная статистика, бар-чарт по дням |
| **Станция** | Детальная информация: график за 24ч, история, рекомендации, безопасно ли находиться рядом |
| **Уведомления** | Список предупреждений о превышении нормы (красная маркировка для критичных) |
| **Настройки** | Тёмная/светлая тема, выбор языка (RU/KK/EN), push-уведомления, интервал обновления данных |

## 4. Дизайн

- Glassmorphism-карточки (`expo-blur` + полупрозрачные фоны)
- Мягкие тени, закругления 20–30px
- Плавные press-анимации через `react-native-reanimated`
- Цветовая индикация статусов, согласованная по всему приложению
- Поддержка тёмной/светлой темы с автоматическим определением системной темы при первом запуске

## 5. API Reference (backend)

| Метод | Путь | Описание |
|---|---|---|
| GET | `/stations` | Список всех станций с текущими показаниями |
| GET | `/stations/:id` | Детали станции + история за 24ч + рекомендации |
| GET | `/statistics` | Средние/макс/мин значения + недельная статистика |
| GET | `/alerts` | Активные предупреждения (dangerous/critical) |

## 6. Возможные доработки после хакатона

- Реальные датчики радиации / интеграция с открытыми API (например, EPA RadNet, Safecast)
- WebSocket для live-обновлений вместо polling
- Настоящие push-уведомления (Expo Notifications) при превышении порога
- База данных (PostgreSQL/MongoDB) вместо JSON-моков
- Кэширование и офлайн-режим (AsyncStorage) для последних данных
